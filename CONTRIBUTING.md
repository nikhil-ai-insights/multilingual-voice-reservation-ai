## ⚙️ Getting Started

### 1️⃣ Fork the Repository
Click the **Fork** button on the top-right of the repository page.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/nikhil-ai-insights/<your-repo-name>.git
cd <your-repo-name>
```

### 3️⃣ Create a Branch

```bash
git checkout -b feature/your-feature-name
```

## 🛠️ Development Guidelines

To maintain quality and scalability across the project:

- Keep workflows **clean, modular, and well-structured**
- Use **clear naming conventions** for all nodes
- Ensure flows are easy to read and understand
- **Avoid breaking** existing functionality
- Build **reusable and scalable** logic wherever possible

---

## 🤖 AI & Voice Guidelines

When working on AI or voice-related features:

- Use **clear and structured prompts**
- Ensure responses feel **natural and human-like**
- Avoid repetitive or robotic replies
- Maintain **multilingual support** throughout
- Keep conversations **concise and contextually relevant**

---

## 🧪 Testing Checklist

Before submitting your contribution, verify the following:

- [ ] Incoming call handling works properly
- [ ] AI correctly understands user intent
- [ ] Reservation booking flow works end-to-end
- [ ] Rescheduling & cancellation flows work
- [ ] Real-time availability checking is accurate
- [ ] Email & SMS notifications are sent correctly
- [ ] Database updates reflect all changes
- [ ] No broken or duplicate flows exist

---

## 📝 Commit Message Guidelines

Use clear, standard commit message formats:

```bash
feat: add multilingual support enhancement
fix: resolve reservation booking issue
docs: update README with setup instructions
refactor: improve intent routing logic
test: add test case for cancellation flow
```

| Prefix | When to Use |
|--------|-------------|
| `feat` | New feature or enhancement |
| `fix` | Bug fix |
| `docs` | Documentation changes only |
| `refactor` | Code restructuring without behavior change |
| `test` | Adding or updating tests |

---

## 🚀 Submitting a Pull Request

**1. Push your changes:**

```bash
git push origin feature/your-feature-name
```

**2. Open a Pull Request** on the main repository.

**3. In your PR description, include:**
- 📌 What you changed
- 💡 Why you changed it
- 🖼️ Screenshots or workflow explanation (if applicable)

> PRs with clear descriptions and tested changes are reviewed and merged faster!
