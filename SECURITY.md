🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly and privately.

📩 How to Report
Email: (add your email here)
GitHub: https://github.com/nikhil-ai-insights
Do NOT open public issues for security-related vulnerabilities
📝 What to Include in Your Report

Please include the following details:

Description of the vulnerability
Steps to reproduce the issue
Potential impact and severity
Proof-of-concept (if applicable)
Suggested fix (optional)
⏱️ Response Timeline

We aim to:

Acknowledge reports within 48–72 hours
Investigate and validate the issue
Provide a fix within 5–10 days depending on severity
🔒 Security Best Practices

This project involves AI, voice processing, and customer data. Please follow these guidelines:

🔑 Credentials & API Keys
Never commit API keys, tokens, or secrets
Use environment variables or n8n credential manager
Rotate credentials regularly
Restrict access to sensitive services
📞 Voice & Call Security
Secure webhook endpoints (HTTPS only)
Validate incoming requests from telephony providers
Prevent unauthorized call triggering
Log and monitor suspicious activity
📧 Notifications (Email & SMS)
Avoid misuse for spam or phishing
Follow communication regulations (GDPR, etc.)
Secure API integrations with proper authentication
📊 Data Protection
Store only necessary customer data
Avoid sensitive personal information
Restrict access to Google Sheets / database
Apply least-privilege access control
⚠️ Known Risks
Misuse for spam calls or automated abuse
Exposure of API keys or credentials
Unauthorized access to reservation data
Weak validation of user input
🧰 Recommended Security Setup

We strongly recommend:

🔐 Enable 2FA (Two-Factor Authentication) on:
GitHub
Google accounts
Cloud services
🔒 Use:
OAuth2 authentication
Secure API endpoints
Access-controlled databases
🚫 Out of Scope

The following are NOT considered security vulnerabilities:

Feature requests
UI/UX improvements
Minor bugs without security impact
Documentation updates
